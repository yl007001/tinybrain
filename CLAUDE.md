# TinyBrain 项目 Claude Code 配置

## 会话开始时自动执行

每次新会话开始时，执行以下操作：

1. **读取项目状态**：检查 `PROJECT-STATUS.md` 了解当前进度
2. **读取 Bug 记录**：检查 `docs/开发日志/` 目录下的最新 Bug 记录文件
3. **记录会话**：在 `E:\1\doc\{hour}\` 目录下创建会话记录文件

## 会话结束时自动执行

每次会话结束前（或用户提醒时），执行以下操作：

1. **更新会话记录**：补充本次会话完成的工作和待完成事项
2. **记录 Bug 修复**：如果修复了 Bug，更新 `docs/开发日志/Bug修复记录-XX.md`
3. **提交代码**：提醒用户是否需要 git commit 和 push

## Bug 记录规范

- 文件命名：`Bug修复记录-01.md`、`Bug修复记录-02.md`（每 10 个 Bug 换新文件）
- 位置：`docs/开发日志/` 目录
- 格式：每个 Bug 包含现象、原因、解决、文件四个部分

## 会话记录规范

- 文件命名：`{日期}_会话记录.md`（如 `2026-06-11_会话记录.md`）
- 位置：`E:\1\doc\{hour}\` 目录
- 内容：会话主题、完成的工作、待完成事项、技术要点

## 文档目录结构

```
docs/
├── 核心技术文档/      ← Spring Boot、MySQL、RAG、Agent 基础
├── 进阶功能文档/      ← Skill、MCP、Agent 深度
├── 工程化文档/        ← 可观测性、Docker、CI/CD
└── 开发日志/          ← Bug 修复记录、开发日志
```

## 重要约定

- 启动项目用 `java -jar` 而不是 `mvn spring-boot:run`
- H2 开发环境用文件库：`jdbc:h2:file:./data/tinybrain`
- 前端 API 超时设置为 120 秒（Agent 调用多个工具需要时间）
- Git 推送需要配置代理：`git config http.proxy http://127.0.0.1:7897`
