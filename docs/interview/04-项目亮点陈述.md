# 项目亮点陈述

## 30秒电梯演讲

> "我独立开发了 TinyBrain——一个基于 Spring Boot 3.x 的个人 AI 知识引擎。
> 它集成了 RAG 检索增强生成和 Agent 智能体能力。
> 核心技术栈：Java 17、Spring Cloud Alibaba、MyBatis-Plus、Resilience4j、
> 以及 IVF 近似最近邻搜索算法。项目已在 GitHub 开源，获得 Apache 2.0 许可。"

## 2分钟详细陈述

### 1. 项目背景
"这是一个覆盖 Java 后端全链路的技术项目，从 Spring Boot 基础框架一直延伸到 AI 大模型集成。
选择这个方向是因为 RAG 和 Agent 是目前业界最热门的 AI 工程化方向。"

### 2. 技术难点与解决方案

**难点 1：向量搜索性能**
"最初使用暴力全量搜索，时间复杂度 O(n)。我调研了业界方案（FAISS、Milvus），
最终在项目中实现了 IVF 倒排索引——通过 K-means 聚类将向量分桶，
搜索时只探测最近的 4 个桶，将搜索复杂度降到 O(√n)。实测 10 万条向量下
搜索速度提升约 50 倍，召回率保持 90% 以上。"

**难点 2：LLM API 的稳定性**
"LLM API 是外部依赖，可能超时或报错。我引入了 Resilience4j 的熔断器、
重试和限流器三层保护——当 API 连续 3 次失败时自动熔断 10 秒，
避免故障级联；同时用熔断降级机制返回友好提示，用户体验不受影响。"

**难点 3：检索质量**
"简单的向量检索在模糊问题上效果差。我实现了三级优化：
1）查询改写——用 LLM 把模糊问题改清晰；
2）IVF + 暴力搜索融合——保证召回率；
3）批量嵌入——100 个分块只需 1 次 API 调用。"

### 3. 工程化能力
"AOP 实现接口耗时监控、Docker Compose 一键部署全套基础设施、
GitHub Actions CI/CD 流水线、Prometheus + Grafana 指标监控、
Zipkin 全链路追踪、JSON 结构化日志配合 ELK 采集。"

### 4. 成果
"项目从零到一全部独立完成，覆盖 7 个 Maven 模块、4,500+ 行 Java 代码、
2,000+ 行前端代码，56 个测试用例全部通过。开源代码质量通过 SpotBugs 和 Checkstyle 检查。"

## 简历上的写法

### 项目经历：TinyBrain 个人 AI 知识引擎 (2026.05 - 至今)

**技术栈：** Java 17 / Spring Boot 3.2 / Spring Cloud Alibaba / MyBatis-Plus /
MySQL / Redis / Elasticsearch / Resilience4j / Docker / Prometheus / Grafana

**项目描述：** 独立开发的全链路 AI 知识管理平台，集成 RAG 检索增强生成与 Agent 智能体。

**核心贡献：**
- 设计并实现 IVF 倒排索引（K-means 聚类 + 多桶搜索），将向量检索速度提升 50 倍
- 构建三级容错体系（Resilience4j 熔断 + 重试 + 限流），保证 LLM API 调用稳定性
- 实现查询改写 + 混合搜索 + 批量嵌入的 RAG 优化管线，提升检索质量
- 搭建 Spring Cloud Gateway + Nacos 微服务架构 + Docker Compose 一键部署
- 配置 Prometheus/Grafana 监控大盘和 Zipkin 分布式追踪
- GitHub Actions CI/CD 流水线 + 56 个自动化测试用例
- 编写 Vue 3 + Element Plus 前端界面（登录、文档管理、RAG 问答、Agent 对话）
